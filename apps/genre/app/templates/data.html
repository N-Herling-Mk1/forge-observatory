Data handling & leakage guards
How GTZAN becomes tensors without leaking — the representations, the track-level split that prevents segment leakage, and the invariants that keep it honest.

1 Data handling

Each 30-second clip exists in several representations; one loader serves all of them through the same split + id logic, so tabular, image, and fused share one source of truth. The scaler and label map are fit on the train split only — computing μ/σ over the whole table before splitting is a quieter form of leakage.

One loader: load(representation, split_strategy, seed, ratios, standardize) → self-describing arrays + label map + train-fit scaler + split provenance.

What consumes these tensors — the dual encoders. BEARDOWN reads each track in two modalities at once, so the model sees timbre/time structure and engineered audio statistics:

Spectrogram → Spec CNN
The image tensor ([224,224,1]) enters a compact CNN — conv → ReLU → pool → flatten → dense — projecting the mel-spectrogram into a latent embedding. Deterministic.
Tabular → MLP
The 58-feature vector enters a small MLP (ReLU + structured dropout), projecting the engineered statistics into a second embedding. Deterministic.

The two embeddings fuse (concat / gated) and feed a probabilistic Bayesian head (Flipout / MC-dropout) → softmax over 10 genres plus an uncertainty estimate. The fused representation above is exactly the dual input this head expects — image and tabular aligned 1:1 per track. (Full architecture diagram lives on the Experiment panel.)

2 Leakage guards

The 3-second table explodes each song into ~10 near-identical segments. Shuffle those 9,990 rows randomly and segments of the same song land in both train and test — the model recognizes the song, not the genre, and accuracy inflates (the Sturm-2013 GTZAN artifact). The guard assigns whole tracks, never segments, stratified per genre so every split still holds all ten genres.

3 Split tests

The guard is only worth trusting if it's enforced. These invariants run in _shared/tests/test_splits.py; the load-bearing one is no-track-overlap — if it goes red, leakage is back.
